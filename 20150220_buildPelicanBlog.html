架pelican blog小記

2015-02-20, 周五

架一個pelican blog最重要的就是會用google就行啦~
當然是簡單的那種，跟這裡一樣!! 基本上我是用windows來架，感覺會比其他的作業系統複雜一點，但我也沒比較過=___=，不是太困難啦~有點耐心就行了。

google一下就有看不完的資料啦!! 譬如說:

官方的文件
Build a blog using the Pelican static site generator
Host a blog on GitHub Pages
嘗試一下 Pelican
Github pages和Pelican搭建靜態博客
How I built this website, using Pelican

他們都說了相當清楚，結束!!

...但為了練習一下markdown怎麼用，所以來稍微打一下。我的環境是windows 7還有Python 3.4。在github上頭架一個pelican部落格大致分做以下步驟:
1. 安裝python：跑pelican用的
2. 安裝及設定pelican：組合html、css用的
3. 開始寫撰寫blog：輸出及測試
4. 美化一下blog：設定基本資料及主題外觀
5. 將blog發佈到github上(請先參考別人的方法！這裡我碰到困難，是用別種方法上傳...)

1.安裝python

先到python的官網下載囉~裏頭有分做3.4跟2.7兩種版本。這兩個版本有些指令並不相容，詳情請google，我也不是很清楚所以就直接安裝了3.4版。接著要來設定python指令的路徑，不然電腦不知道python放哪裡，請參考此篇"Python於MS-Windows的命令列路徑設定"。另外建議安裝virtualenv，為了不要讓不同的函數庫互相衝突。

詳細一點可以看我這篇"在windows下設定簡單的python編寫環境"的設定。

2.安裝及設定pelican

2.1 安裝pelican:

pelican說明文件其實已經寫得蠻清楚的，打開windows指令列然後輸入以下，結束!!!

pip install pelican markdown
2.2 設定pelican project:

在指令列移動到你想要放pelican blog的地方，然後輸入

pelican-quickstart

他會囉哩八說地問一堆問題，回答完就大功告成啦~~
(如果有缺甚麼套件就使用pip install XXXX安裝。)

3.開始撰寫blog

3.1 簡單的第一篇:

此時你的資料夾裡面會有一叫做"content"的資料夾，這個地方就是要擺文章的地方。之前的一堆問題他會問你要用哪一個格式發表文章，我是選擇用markdown。這個格式非常簡單，跟一般text打起來差不多，花個十分鐘把這個看完，接下來練習打一篇(就是現在這一篇)就差不多了。接下來在這個"content"資料夾新增一個檔案，比方說"firstTime.md"，這就是一篇blog！裡面可以打

Title: 嗨~~第一篇
Date: 2015-02-14 00:00
Authors: 三重金城武

今天是情人節~我學會用大嘴鳥部落格。真幸福~~~

接下來就開始用pelican把上頭的文章轉換成html、css囉~在指令列裏頭輸入

pelican content

他會把輸出的網頁會放在"output"的資料夾裡。

3.2 預覽一下:

接下來我們稍微預覽一下這個blog長甚麼樣子@@，在官方文件的quickstart中是說指令列移至"output"的資料夾，輸入python -m SimpleHTTPServer。不過你若是灌python3.4，請換另一個不simple的server~~

python -m http.server

接著打開你的瀏覽器輸入 http://localhost:8000/ 就可以看到blog的誕生啦!!!恭喜恭喜~
這個指令會讓你沒辦法編輯...指令視窗被拿去顯示伺服器的資訊，按ctrl+c可以終止。但這樣一直進進出出很累，其實可以寫一個python的小程式丟在output的資料夾裡面，要預覽時點開他還可以繼續編輯，這樣方便多了。

#!/usr/bin/env python
# -*- coding: utf-8 -*- #
from __future__ import unicode_literals

#網站的基本資料，如同字面所述
#其中的SITEURL，若沒有自己的domain先不要設，
#後頭要設成github repository的發佈網址
AUTHOR       = 'dekko Shen'
SITENAME     = "blogblogblog...blog"
SITEURL      = 'http://blog.dekkoshen.com' 
DEFAULT_LANG = 'zh'
TIMEZONE     = 'Asia/Taipei'
DATE_FORMATS = {
    'en':('usa','%a, %d %b %Y'),
    'zh':('chs','%Y-%m-%d, %a'),
}

#連結的資料夾
PATH         = 'content'     #放文章的資料夾名稱
STATIC_PATHS = ['images']    #內部連結的資料夾名稱，要放在content裡

# 每篇文章的網址設定，'slug'是待會要設在文章裡面的標籤，會作為網頁名稱
ARTICLE_SAVE_AS = '{slug}.html'
ARTICLE_URL     = '{slug}.html'

#要選擇哪一個外觀主題
THEME = "./myPelicanTheme/basic/"
DEFAULT_PAGINATION = 10         #一頁要放幾篇文章

# 設定feed的地方
FEED_DOMAIN           = None
FEED_ALL_ATOM         = None
FEED_ATOM             = None
CATEGORY_FEED_ATOM    = None
TRANSLATION_FEED_ATOM = None
AUTHOR_FEED_ATOM      = None
AUTHOR_FEED_RSS       = None

# Blogroll
LINKS = (
         ('我的攝影網站', 'http://dekkoShen.com'),
         )

# Social widget
SOCIAL = (('facebook', 'https://www.facebook.com/dekko.shen'),
          ('instagram', 'http://instagram.com/dekkoshen/'),
          )

DISQUS_SITENAME  = 'dekkosblogblogblog'  #DISQUS留言板的代號

# 如果要用python http.server測試，最好先將他 uncomment起來，
# 不然一按其他的連結會自動跳成上面所設定的'SITEURL'
#RELATIVE_URLS = True

關於其他的設定可以參考官方文件的Settings，有相當多啊！

4.2 文章的基本設定:

因為我是選擇用markdown的格式，如果要寫一篇blog，我會新增一個檔案'fileName.md'到'content'的資料夾裡。這個檔案會像這種格式

Title: 這可是另一篇
Date: 2015-02-14 23:59
Slug: 20150214_moon
Category: 有的沒的
Tags: 心情
Authors: 三重金城武
Summary: 今日情人

巧克力怎麼不溶我手啊啊啊!!!!!

前面那堆metadata的功用就如同字面所述，而Slug就是在設定這篇blog的網址名稱，所以他看起來會像是"http://www.XXXX.com/20150214_moon.html"。注意喔！這個slug必須是唯一的，不然若兩篇文章是同一個slug會不知道要選哪一篇(我也沒試過)。

4.3 選擇一個主題吧:

接下來就要來美化blog的頁面啦!剛裝好的pelican只有兩個主題可以選，他們會放在Python34/Lib/site-packages/pelican/themes裡面，不過看起來不怎麼樣啊！但我也不會製作theme，所以只好抓別人的來改一改，嘿嘿嘿。pelican的github裡面提供了許多的theme可以選擇，右邊有個"download zip"可以先把他們全部抓下來，而從這裡可以預覽一下剛剛那些theme大概長甚麼樣子。選擇好theme，就把這個資料夾的位置設定在"pelicanconf.py"中"THEME"裡面就大功告成啦!

這時候可以把blog用pelican content輸出看看，但有可能會出現這樣的錯誤訊息

CRITICAL: unorderable types: Article() < Article()

應該是有些theme的templates有誤。可以打開theme資料夾中的"tags.html"，裡面會有類似的一段程式

<ul class="square">
    {% for article in articles|sort %}
    <li><a href="{{ SITEURL }}/{{ article.url }}">{{ article.title }}</li>
    {% endfor %}
</ul>

將裡面的sort刪掉變成{% for article in articles %}，應該就可以了。如果不行試著把cache清掉

pelican --ignore-cache content/

再跑一次試試看。

5.發佈在github上頭

5.1 安裝git及申請github帳號:

git是版本控制系統，github則是分享git計畫的網路平台，可以把東西丟在上頭，不過要用git去控制才行。而擺在github上面的東西必須要公開，不然就要收錢啦！這很適合拿來架靜態的網站，反正本來就是要公開的東西...

先申請一個github帳號，但因要用github就必須要安裝git，不過很簡單，github網站已經有提供在windows的控制軟體，他會自動幫你安裝好git了。

5.2 準備發佈:

這裡請先參考別人作法！像這篇"Github pages和Pelican搭建靜態博客"。

因為我沒辦法順利執行這個指令

ghp-import -b master -p output

會出現這樣的錯誤訊：息TypeError: Type str doesn't support the buffer API
目前我查不出來該怎麼處理，不知是不是python3.4的問題？所以我先用別種方法來上傳blog，以後再來研究哪裡出錯好了...

我的方法麻煩很多，首先將指令列移動到"output"的資料夾，然後打上

git init

此時會多一個".git"的隱藏資料夾，用來儲存不同版本的差異。因為現在是在master的分支上，如果要在github上發佈網頁必須要在"gh-pages"的分支，所以我們要切過去

git checkout -b gh-pages

這時候指令列的位置後面應該會寫著(gh-pages)。接下來我們要把這個資料夾的東西包裝成一個新的版本(commit)，準備要發佈到github上頭。步驟如下:

git add -A
git commit -m "你的註解"

第一行git add -A是把所有的資料都丟到包裝區；第二行的git commit則是把包裝區的東西包成一個新版本放到repository裡面。它的結構像是

因為最終的目的是要上傳到github，所以我們先到github開一個新的repository。但注意，要是你沒有自己的domain，這個repository名稱將會決定最後這個blog的網址喔~會是這個樣子 

http://userName.github.io/repositoryName

注意，若沒有自己的domain，要將pelicanconf.py中的SITEURL修改成上述的網址。建立好以後，應該會看到一個連線網址 https://github.com/userName/repositoryName.git，我們要用它來跟本機做連線，所以先做名叫"origin"的一個遙控器

git remote add origin https://github.com/userName/repositoryName.git

現在就可以來發布我們的部落格啦！只要打上

git push origin gh-pages

就大功告成了！可以到http://userName.github.io/repositoryName看看是否有看到自己的blog。

5.3 之後的發佈流程:

  1. 編輯文章，然後執行

    pelican content/

    輸出到"output"的資料夾，再預覽及修改。

  2. 修改完後，將指令列移至"output"的資料夾，依序輸入

    git add -A
git commit -m "你的註解"
git push origin gh-pages
